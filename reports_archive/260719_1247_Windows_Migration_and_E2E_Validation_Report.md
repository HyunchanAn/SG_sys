### 1. 개요 (Executive Summary)

본 보고서는 Mac 환경의 극심한 메모리 부족(OOM) 한계로 인해, 기존 MSA 15개 모듈 및 통합 파이프라인을 64GB RAM과 RTX 5080이 장착된 Windows 메인 워크스테이션 환경으로 재이관(Migration)하고, 통합 E2E 테스트를 성공적으로 완수한 결과를 기술합니다.

윈도우 도커 고유의 UNC 경로(원격 네트워크 볼륨) 바인드 마운트 한계를 파워쉘 스크립팅 기반의 '수동 DB 스트리밍' 로직으로 영리하게 우회하였으며, 파워쉘 특유의 비정상 종료(NativeCommandError) 버그를 무시 정책(`Continue`)으로 완벽히 방어하여 전체 파이프라인의 무중단 런타임을 확보했습니다.

### 2. Windows 환경 최적화 및 런타임 우회 (Infrastructure & Workaround)

#### 2.1. Docker 환경 롤백 및 GPU 통제권 복구
* **Windows 인프라 세팅**: Mac 환경을 위해 삭제되었던 컨테이너 GPU 할당 설정(`deploy: reservations: devices: driver: nvidia`)을 `docker-compose-windows.yml`에 복구하여, 추론 모듈(`SG_proj_001`, `004` 등)이 윈도우의 RTX 5080을 온전히 100% 활용하도록 설계했습니다.
* **네임드 볼륨 격리**: `postgres_data_vol`을 도커 내부 네임드 볼륨으로 지정하여 원격 네트워크 드라이브를 강제로 호스트 볼륨으로 매핑할 때 발생하는 윈도우 파일시스템 인식 오류를 방지했습니다.

#### 2.2. UNC 경로 바인딩 에러 우회 및 수동 DB 복구
* **이슈**: 초기화된 빈 DB 컨테이너에 백업 파일(`init.sql`)을 마운트시키려 했으나, 윈도우 도커 데몬은 원격 UNC 경로의 단일 파일(File Bind Mount) 인식조차 거부하여 컨테이너 생성이 멈췄습니다.
* **해결 (PowerShell 스트리밍)**: 마운트 구문을 삭제하고 우선 빈 DB로 부팅시킨 뒤, 15초 대기 후 파워쉘 자체 권한으로 원격 경로의 `init.sql`을 읽어 컨테이너 내부의 psql 쉘로 강제 주입하는 스트림(`Get-Content ... | docker exec ...`) 복원 로직을 작성했습니다. 이를 통해 모든 기존 데이터가 완벽히 살아났습니다.

#### 2.3. PowerShell 파이썬 경고 크래시 우회
* **이슈**: 딥러닝 추론 컨테이너(PyTorch 등) 테스트 종료 시 발생하는 단순 C++ 스레드 경고문(Stderr)을 파워쉘이 치명적 오류(`NativeCommandError`)로 오인하여 전체 반복문 프로세스를 강제로 Exit 시키는 윈도우 고질적 버그가 발생했습니다.
* **해결**: 파워쉘 스크립트 최상단에 `$ErrorActionPreference = "Continue"` 정책을 강제 주입하여, 허위 경고를 무시하고 15개 전체 디렉토리를 멈춤 없이 순회하도록 방어했습니다.
