- **마이크로서비스(MSA) 오케스트레이션 및 비동기 API 통신 전환 (PoC 완수)**
  - `SG_sys/docker-compose.yml`을 신설하여 `004_db`, `004_api`, `001_api`를 하나로 묶는 최상위 컨테이너망(`sg_network`) 구축 (Root Orchestration).
  - `SG_proj_001` 모듈 내에서 억지로 로컬 `004` 경로를 임포트(`sys.path.append`)하던 하드코딩을 제거하고, `httpx` 비동기 클라이언트를 이용해 `http://sg_proj_004_api:8000/adherends` 데이터를 호출하도록 구조적 리팩토링 완료.
  - 통합 테스트 스크립트(`run_all_tests.sh`)를 개선하여, MSA 모듈들은 독립 컨테이너 내부에서 격리 구동(`docker-compose exec`) 되도록 테스트 인프라 고도화.
  - 향후 내부망 API 통신 시 별도의 API Key 인증(Auth) 방식으로 전환할 수 있는 확장성을 염두에 둠.

- **통합 시스템 보안 및 메모리 최적화 (untracked_vulnerability_report_proposal 조치)**
  - **모듈 파편화(Version Fragmentation) 원천 차단**: 프론트엔드 통합 모듈(`SG_integration_step2`, `step3`) 저장소 내부에 Submodule/Copy&Paste 형태로 중복 방치되던 타 모듈(`004`, `001` 등)의 코드를 완전 삭제하고, 중앙 집중형 MSA REST API를 호출하도록 스왑.
  - **보안 토큰 인증 중앙화**: 각 모듈에 파편화 및 하드코딩될 위험이 있던 민감 키(`OPENAI_API_KEY` 등)를 오케스트레이터의 `SG_sys/.env` 한 곳으로 격리하고, `docker-compose.yml`을 통해 런타임 환경 변수로만 하위 컨테이너에 안전하게 주입.
  - **VRAM OOM(Out of Memory) 방어**: 딥러닝 비전 연산이 포함된 `002`, `003`, `007` 모듈의 FastAPI 엔드포인트 종료 시점에 GC 호출 및 VRAM 반환 미들웨어 탑재. 
    * `torch.cuda.is_available()`와 `torch.backends.mps.is_available()`를 동시 체크하여, NVIDIA GPU 서버와 현재 Apple Silicon (M2 Pro) 맥북 환경 모두에서 동작하는 **크로스 플랫폼 최적화** 적용.

- **통합 검증 완료**
